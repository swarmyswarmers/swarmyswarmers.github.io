Milestone Explanations

Assemble PoC: Put together a car. This includes prepping the correct software too â€“ Python, openCV, Bluetooth, etc.

Remote Control via BT Keyboard: Use Bluetooth (probably Python via pyBluez library) to send drive commands to the car. Take the commands and put them into a queue. Each command should be a direction along with a magnitude between 1 and 10. Examples: R1, L4, R10.

Spoof Leader Communication: Connect from stationary RPi to car and send commands simulating car-to-car communication. Use Python scripts to automate the sending so we can use them later for testing.

Anti-Collision: Install colored blocks of cardboard to the back of each car so that the camera/openCV can recognize them. Train openCV on the colored blocks and link openCV results to command softening protocols.

Command Softening: Sometimes a will be sent to a car but the car needs to avoid collision at all costs so we should soften commands that would normally result in a collision. This means that if the car detects an object in the path of the command that the magnitude of the command should be reduced by a factor dependant on how near the object is.

Assemble 2nd car: see Assemble PoC

Election State: In RAFT-like fashion, each car should start in a follower state then wait for a random timeout before going to the election state. The car will be need greater than N/2 confirmed votes to become a leader and start broadcasting commands.

Assemble 3rd Car: see Assemble PoC

Leader death and reconnection: This is basically a test of the state change timeout, election protocol, and re-introduction of another car as a follower. This should all follower the RAFT-like protocol.

Ad-hoc car pickup: This is an extension of the Leader death and reconnection milestone.

Bug squashing: git dem bugs.
